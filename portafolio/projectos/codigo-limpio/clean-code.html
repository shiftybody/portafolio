` Clean Code

Escribir código limpio significa asegurarse de que el código sea

fácil de leer, mantener y extender. Los nombres de variables y funciones

ser claros y descriptivos, reflejando la intención del código sin necesidad de comentarios adicionales.

Una de las claves de Clean Code es aplicar el principio de Single Responsibility, donde cada método o clase debe tener una única responsabilidad. Esto ayuda a reducir la complejidad y facilita los cambios en el futuro.

Los comentarios deben ser usados con moderación, ya que el código bien escrito no debería necesitar demasiada explicación externa. La preferencia siempre debe ser autoexplicativo.

Evitar las funciones largas es fundamental en Clean Code, ya que el código debe ser modular y fácil de probar. Una función ideal debería cumplir con el principio de small functions, manteniendo su tamaño reducido para realizar una sola tarea
